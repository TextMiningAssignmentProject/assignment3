"객체는 실체의 객관적인 측면"이다. 여기에서 예를 들어서 내가 현재 블로그에 글을 쓰고 있는 도구인 키보드는 실체이다. 이 키보드를 보았을 때 외관적인 특징과 그 외의 특징을 단순하게 나열해보자면 다음과 같다.키보드의 단순한 특징1. 사각형으로 되어있다.2. 그 위에 여러 사각형들이 있다.3. 그 위에 있는 사각형들을 누르면 소리가 난다.4. 긴 줄과 연결되어 있다. ( 무선 키보드 제외 )같이 기숙사 생활을 하고 있는 키보드를 모르는 친구나 선배한테 키보드를 가져와 달라고 한다면, 위와 같은 단순한 특징들을 설명하여 키보드가 무엇인지 알 수 있게 해줄 수 있다. 이렇게 실체의 형태를 추상화( 단순하게 말해서 큰 특징들로만 일반화 시킨 것 ) 하면, 추상화된 실체를 컴퓨터에서 모방할 수 있게 된다. 이때 이러한 형태를 객체라고 하며, 컴퓨터에서 객체의 특징과 동작 등등 을 구현하여 객체들끼리의 상호 작용시켜서 만들 수 있다고 한다면 객체 지향 형식으로 만들었다고 볼 수 있다.​중요한 것은 객체 지향 언어는 객체를 만들기 위해 설계도가 필요하며 이러한 설계도를 클래스라고 한다. 이때 클래스를 가지고 객체를 생성할 수 있으며 객체와 객체를 연결하여 목적에 맞는 프로그램을 만들어낼 수 있다. 근데 자바가 아무리 작은 프로그램이라도 객체를 만들어서 사용해야 한다는 점이 있다. 그 이유는 처음부터 객체를 고려하여 설계된 언어이기에 객체 지향 언어가 가져야 할 캡슐화, 상속, 다형성 기능들을 지원하고 있다.​객체 지향은 절차 지향 이후에 나온 개념이기 때문에 절차 지향에서의 단점들을 보완할 수 있는 장점이 있을 것이다. 하지만 학교에서 자바를 배우면서 크게 느껴보지는 못했다. 아마도 큰 프로젝트나 혹은 자바의 객체 지향 형식으로 만들어내는 디자인 패턴들이 유지 보수나 실세계에서 살고 있는 우리에게 직관적으로 알 수 있고 표현할 수 있어서 객체 지향을 사용하는 것으로 이해하고만 있다.​3. 메모리 관리2020년 1월부터 2월까지 금전적인 이유로 아시는 대표님에게 프로젝트를 받아서 진행했던 경험이 있었다. 그때 프로그램을 하던 도중 핵심적으로 동작하는 프로세스가 메모리 누수(Memory Leak)를 일으켰다. 오픈 소스를 가지고 했던 것인데, 해당 오픈소스가 워낙 광범위해서 내가 직접 코드를 수정하다간 지정된 날까지 끝을 못 낼 것 같다는 의견에 동의하면서 다른 핵심 기능을 만들어야 했었다. 우리가 잘 알듯이 메모리는 RAM이고 프로그램이 프로세스로 동작하려면 해당 RAM에 충분한 용량이 있어야 한다. RAM에 충분한 용량이 없다면, 프로세스가 정상적으로 동작하지 않고 오히려 운영체제가 해당 프로세스를 강제적으로 종료시키는 경우도 생긴다.​C++은 객체지향 언어 중에 하나이지만, 메모리 관리를 개발자가 직접 해줘야 한다. 이것이 무슨 이야기냐면, 객체를 생성하면 생성된 객체는 메모리에 할당이 된다. 즉 RAM(주 기억장치)의 용량을 어느 정도 쓴다는 것인데, 객체가 무수히 증가하는 형태의 프로그램을 짜게 된다면 위와 설명한 것과 같이 메모리 누수로 인하여 운영체제에 의해서 프로세스가 종료되거나 정상적으로 동작되지 않을 것이다. 그렇기에 C++에서는 객체를 생성 이후, 필요 없는 객체는 언제 제거할 것인가를 생각하여 프로그램을 짜야 하는데, 자바 같은 경우 객체 생성 시 자동적으로 메모리 영역을 할당하고, 사용이 완료되었다고 판단되면 가비지 컬랙터(GC)를 실행시켜 자동적으로 필요하지 않은 객체를 제거시켜준다. 하지만 이렇게 동작한다고 해서 가비지 컬렉터를 굳게 믿어서는 안 된다. 가비지 컬렉터도 사람이 짠 프로그램이기에 자바에서도 메모리 누수가 일어날 수 있다. ( 실제로 경험담은 아니지만 커뮤니티에서 읽었음 ) 그렇기에 자바 개발자는 GC에 대한 학습도 필요하며, GC를 튜닝을 하거나 사용 방법을 학습하여 어떻게 프로그램을 효율적으로 동작하게 할 방법에 대해서도 알아야 한다. 어찌 되었든, 가비지 컬렉터로 인하여 개발자는 메모리 관리에 대해서 크게 신경을 쓸 필요는 없다는 것이고 핵심적인 기능에 대한 코드 작성에 몰입할 수 있게 도와준다.​이외에 다양한 장점들이 있다.​- 멀티쓰레드 구현 가능- 동적로딩 지원- 오픈소스 라이브러리가 풍부​JDK - JRE - JDKJVM ( Java Virtual Machine )​JVM은 자바의 특징 중에 이식성을 맡고 있으며, HOST OS 대신에 자바 프로그램을 실행시키는 가상 운영체제 역할을 한다. JVM 같은 경우에는 운영체제 ( HOST OS ) 에 따라서 프로그램을 실행하고 관리하는 방법이 다르기에, 윈도우에서 사용한 JVM을 리눅스에 올려서 사용한다면 윈도우에서 짠 자바 프로그램을 사용하지 못한다. 즉, JVM은 각 OS에 맞춰서 만들어진 것이 다 다르기에 그에 맞게 설치해서 사용해야지만 자바의 특징인 이식성을 보장할 수 있다.​자바 프로그램과 JVM 동작 방식에 대해서 연관 지어 간단하게 설명해보자면 다음과 같다.​먼저 자바 프로그램 소스 파일을 생성한 후, 도스 창에서 javac.exe 를 실행시키면  바이트 코드 파일이 생성되며, 확장자명은 .class이다. 이때 class 파일을 가지고 JVM 구동기인 java.exe를 실행시키면 JVM에 의해서 현재 운영체제에 맞게 기계어로 번역된 후 프로그램이 실행된다.사진 1 ) 간단 설명​JRE ( Java Runtime Environment)​JRE에는 자바 프로그램 실행에 필요한 라이브러리와 소프트웨어가 포함된다. 무슨 이야기냐면 자바 프로그램을 짠 이후에 java.exe 컴파일러가 동작할 때 기본적으로 동작해야 하는 필수적인 라이브러리가 포함되어 동작한다는 뜻이다.
